Java Collections Framework: Hướng dẫn chi tiết

29/12/2025

Giới thiệu

Java Collections Framework (JCF) cung cấp một kiến trúc thống nhất để lưu trữ và thao tác với nhóm các đối tượng. Nó bao gồm các Interfaces (Giao diện), Implementations (Lớp cài đặt) và Algorithms (Thuật toán). Sử dụng Collection giúp bạn không phải tự viết lại các cấu trúc dữ liệu cơ bản như danh sách lien kết hay bảng băm.

Các Interface chính

Hệ thống Collection trong Java xoay quanh các interface cốt lõi sau:

  1. List: Lưu trữ danh sách có thứ tự, cho phép phần tử trùng lặp.
  2. Set: Lưu trữ tập hợp các phần tử không trùng lặp (duy nhất).
  3. Map: Lưu trữ dữ liệu dưới dạng cặp Key-Value (Khóa - Giá trị).

1. List Interface

ArrayList

  • Đặc điểm: Dùng mảng động (dynamic array) để lưu trữ.
  • Ưu điểm: Truy cập phần tử cực nhanh (O(1)) thông qua index.
  • Nhược điểm: Thêm/Xóa phần tử ở giữa danh sách chậm vì phải dịch chuyển các phần tử khác.
  • Khi nào dùng: Khi bạn cần đọc dữ liệu nhiều hơn là thay đổi cấu trúc.
List<String> list = new ArrayList<>();
list.add("Java");
list.add("Python");
System.out.println(list.get(0)); // Output: Java

LinkedList

  • Đặc điểm: Dùng danh sách liên kết đôi (doubly linked list).
  • Ưu điểm: Thêm/Xóa phần tử cực nhanh (O(1)) ở bất kỳ vị trí nào nếu có reference.
  • Nhược điểm: Truy cập ngẫu nhiên chậm (O(n)) vì phải duyệt từ đầu.
  • Khi nào dùng: Khi ứng dụng cần thêm/xóa phần tử liên tục.

2. Set Interface

Set chuyên dùng để loại bỏ các phần tử trùng lặp.

HashSet

  • Không đảm bảo thứ tự các phần tử.
  • Hiệu năng cao nhất (O(1)) cho các thao tác add, remove, contains.
  • Dùng cơ chế băm (hashing).

LinkedHashSet

  • Duy trì thứ tự thêm vào (insertion order).
  • Chậm hơn HashSet một chút do phải duy trì danh sách liên kết.

TreeSet

  • Sắp xếp các phần tử theo thứ tự tăng dần tự nhiên hoặc theo Comparator.
  • Chậm hơn (O(log n)) do dùng cấu trúc cây (Red-Black tree).
Set<String> uniqueNames = new HashSet<>();
uniqueNames.add("Hào");
uniqueNames.add("Hào"); // Bị bỏ qua
System.out.println(uniqueNames.size()); // Output: 1

3. Map Interface

Map không kế thừa từ Collection interface nhưng là một phần quan trọng của JCF.

HashMap

  • Là Map phổ biến nhất. Lưu trữ Key-Value.
  • Key không được trùng, Value có thể trùng.
  • Cho phép 1 Key là null.

TreeMap

  • Lưu trữ Key-Value nhưng sắp xếp Key theo thứ tự.
Map<String, Integer> scores = new HashMap<>();
scores.put("Hào", 90);
scores.put("Nam", 85);

for (String key : scores.keySet()) {
    System.out.println(key + ": " + scores.get(key));
}

Iterator và Loops

Có nhiều cách để duyệt qua Collection:

  1. For-each loop: Cách gọn nhất và phổ biến nhất.
  2. Iterator: Dùng khi cần xóa phần tử trong lúc duyệt để tránh ConcurrentModificationException.
  3. Java 8 Streams: Cách hiện đại, hỗ trợ xử lý song song và functional programming.
// Iterator example
Iterator<String> it = list.iterator();
while(it.hasNext()) {
    String item = it.next();
    if(item.equals("Xoa")) {
        it.remove(); // An toàn
    }
}

Kết luận

Java Collections Framework là một chủ đề rộng lớn. Việc chọn đúng cấu trúc dữ liệu (ArrayList vs LinkedList, HashSet vs TreeSet) sẽ ảnh hưởng rất lớn đến hiệu năng ứng dụng của bạn. Hãy luôn cân nhắc tính chất của dữ liệu và các thao tác bạn sẽ thực hiện để đưa ra lựa chọn tối ưu.
