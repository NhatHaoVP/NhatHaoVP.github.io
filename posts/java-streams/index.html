Java Streams API: Xử lý dữ liệu hiện đại

29/12/2025

Giới thiệu

Được giới thiệu trong Java 8, Stream API mang đến một cách tiếp cận hoàn toàn mới để xử lý Collections. Thay vì dùng các vòng lặp for hay while dài dòng, bạn có thể viết code theo phong cách khai báo (declarative), ngắn gọn và dễ đọc hơn.

Stream không lưu trữ dữ liệu, nó chỉ chuyển dữ liệu từ nguồn (source) qua một chuỗi các thao tác xử lý (pipeline).

Cấu trúc của Stream Pipeline

Một Stream pipeline gồm 3 phần:

  1. Source: Nguồn dữ liệu (Collection, Array, I/O channel…).
  2. Intermediate Operations: Các thao tác trung gian (filter, map, sorted…). Trả về một Stream mới. Lazy execution (chỉ chạy khi có terminal operation).
  3. Terminal Operation: Thao tác kết thúc (collect, forEach, reduce…). Trả về kết quả hoặc void. Kích hoạt toàn bộ pipeline.

Các thao tác biến đổi phổ biến

1. Filter (Lọc)

Lọc các phần tử thỏa mãn điều kiện.

List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5, 6);
// Lấy số chẵn
List<Integer> evens = numbers.stream()
                             .filter(n -> n % 2 == 0)
                             .collect(Collectors.toList()); 
// Kết quả: [2, 4, 6]

2. Map (Ánh xạ)

Biến đổi mỗi phần tử thành một giá trị khác.

List<String> names = Arrays.asList("a", "b", "c");
// Chuyển thành chữ hoa
List<String> upper = names.stream()
                          .map(String::toUpperCase)
                          .collect(Collectors.toList());
// Kết quả: ["A", "B", "C"]

3. Sorted (Sắp xếp)

names.stream().sorted().forEach(System.out::println);

4. Reduce (Thu gọn)

Gộp các phần tử lại thành 1 giá trị duy nhất.

int sum = numbers.stream().reduce(0, (a, b) -> a + b);

Parallel Streams (Luồng song song)

Java Streams hỗ trợ xử lý song song cực kỳ đơn giản với parallelStream(). Nó tận dụng kiến trúc đa lõi của CPU để tăng tốc độ xử lý dữ liệu lớn.

list.parallelStream().filter(...).collect(...);

Lưu ý: Chỉ dùng Parallel Streams khi dữ liệu thực sự lớn và tác vụ xử lý độc lập với nhau. Nếu dùng sai có thể gây chậm hơn hoặc lỗi thread-safety.

Các sai lầm thường gặp

  1. Tái sử dụng Stream: Một Stream sau khi đã gọi terminal operation thì sẽ bị đóng. Nếu gọi lại sẽ ném Exception.
  2. Thay đổi dữ liệu gốc: Stream API được thiết kế để không thay đổi (mutate) dữ liệu nguồn. Hãy trả về list mới thay vì sửa list cũ.
  3. Quên Terminal Operation: Nếu không có terminal op, các lệnh filter/map sẽ KHÔNG BAO GIỜ CHẠY.

Kết luận

Stream API giúp code Java trở nên thanh thoát và mạnh mẽ hơn rất nhiều. Kết hợp với Lambda Expressions, đây là kỹ năng bắt buộc phải có của một Java Developer hiện đại.
