Lập Trình Hướng Đối Tượng (OOP) trong Java

29/12/2025

Giới thiệu về OOP

Lập trình hướng đối tượng (Object-Oriented Programming - OOP) là một mô hình lập trình dựa trên khái niệm “đối tượng” chứa dữ liệu (thuộc tính) và mã lệnh (phương thức). Java là ngôn ngữ thuần OOP, do đó hiểu rõ 4 nguyên lý cơ bản của OOP là bắt buộc.

4 Nguyên lý cơ bản (The 4 Pillars of OOP)

1. Tính Đóng Gói (Encapsulation)

Đóng gói là việc che giấu dữ liệu bên trong nội bộ của đối tượng và chỉ cho phép truy cập thông qua các phương thức công khai (getter/setter). Điều này giúp bảo vệ dữ liệu khỏi bị thay đổi tùy tiện.

public class BankAccount {
    private double balance; // Dữ liệu private

    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount; // Chỉ thay đổi qua logic kiểm soát
        }
    }

    public double getBalance() {
        return balance;
    }
}

2. Tính Kế Thừa (Inheritance)

Cho phép một class con (subclass) kế thừa các thuộc tính và phương thức từ class cha (superclass). Từ khóa extends. Giúp tái sử dụng mã nguồn (code reusability).

class Animal {
    void eat() { System.out.println("Eating..."); }
}

class Dog extends Animal {
    void bark() { System.out.println("Woof!"); }
}
// Dog có cả eat() và bark()

3. Tính Đa Hình (Polymorphism)

Cho phép một hành động có thể được thực hiện theo nhiều cách khác nhau. Đa hình thường thể hiện qua:

  • Overloading (Nạp chồng): Cùng tên hàm nhưng khác tham số (Compile-time polymorphism).
  • Overriding (Ghi đè): Class con viết lại hàm của class cha (Runtime polymorphism).
// Overriding example
class Cat extends Animal {
    @Override
    void eat() {
        System.out.println("Cat eats fish");
    }
}

4. Tính Trừu Tượng (Abstraction)

Ẩn đi các chi tiết cài đặt phức tạp và chỉ hiển thị những tính năng cần thiết cho người dùng. Thực hiện thông qua Abstract Class và Interface.

  • Abstract Class: Có thể chứa cả phương thức trừu tượng (không có thân hàm) và phương thức cụ thể. Dùng khi các class có quan hệ chặt chẽ “is-a”.
  • Interface: Chỉ chứa hằng số và phương thức trừu tượng (trước Java 8). Dùng để định nghĩa hành vi “can-do”.

Từ khóa this và super

  • this: Tham chiếu đến đối tượng hiện tại. Thường dùng để phân biệt biến instance và biến tham số.
  • super: Tham chiếu đến đối tượng của lớp cha. Dùng để gọi constructor hoặc phương thức của lớp cha.
class Person {
    String name;
    Person(String name) { this.name = name; }
}

class Employee extends Person {
    double salary;
    Employee(String name, double salary) {
        super(name); // Gọi constructor lớp cha
        this.salary = salary;
    }
}

Kết luận

OOP giúp code dễ bảo trì, dễ mở rộng và mô phỏng thế giới thực tốt hơn. Hãy luyện tập việc thiết kế class diagram trước khi bắt tay vào code để áp dụng triệt để các nguyên lý này!
